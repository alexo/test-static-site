Technical Details

Backend Persistence (UTC Conversion)

All datetime values are converted to UTC before being sent to the backend. This ensures consistent storage regardless of user timezone selection.

// Frontend to Backend Conversion
const userSelectedDate = '2026-04-10 14:30'; // User input in America/Los_Angeles
const timezone = 'America/Los_Angeles';

// Convert to UTC for backend persistence
const utcDate = moment.tz(userSelectedDate, timezone).utc();
const backendValue = utcDate.format('YYYY-MM-DD HH:mm:ss'); 
// Result: '2026-04-10 22:30:00' (stored in database)

// Backend to Frontend Display
const dbValue = '2026-04-10 22:30:00'; // UTC from database
const userTimezone = getUserTimezone(); // e.g., 'America/Los_Angeles'

// Convert from UTC to user's timezone for display
const localDate = moment.utc(dbValue).tz(userTimezone);
const displayValue = localDate.format('MMMM D, YYYY HH:mm z');
// Result: 'April 10, 2026 14:30 PDT'

Integration Pattern

const { timezone, setTimezone, 
        timezoneLabel, options } = 
    useTimezone(timezoneOptions);

<TimezoneSelectorTooltip
    timezone={timezone}
    timezoneLabel={timezoneLabel}
    options={options}
    onTimezoneChange={setTimezone}>
    <DatePicker ... />
</TimezoneSelectorTooltip>
